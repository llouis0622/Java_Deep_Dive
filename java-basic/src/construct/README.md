## 4. 생성자

### 1. 생성자  - 필요한 이유
- 객체를 생성하는 시점에 어떤 작업을 하고 싶을때 -> 생성자 이용

### 2. this
- 멤버 변수에 접근하기 위해 `this` 사용
- 인스턴스 자신의 참조값을 가리킴
- 매개변수의 이름과 멤버 변수의 이름이 같은 경우 `this`를 사용해서 둘을 명확하게 구분해야 함
- `this` 생략 가능 : 가까운 지역변수를 먼저 찾고 없으면 그 다음으로 멤버 변수를 찾음

### 3. 생성자 - 도입
- 객체를 생성하는 시점에 즉시 필요한 기능 수행 가능
- 생성자의 이름 = 클래스 이름
- 생성자 반환 타입 없음
- 인스턴스를 생성하고 나서 즉시 호출 -> `new` 명령어 다음 생성자 이름과 매개변수에 맞추어 인수 전달
- 중복 호출 제거 : 객체 생성 + 동시에 생성 직후 필요한 작업 한 번에 처리 가능
- 객체 생성 시 직접 정의한 생성자 존재 -> 직접 정의한 생성자를 반드시 호출
- 생성자 사용 -> 필수값 입력 보장 가능

### 4. 기본 생성자
- 기본 생성자 : 매개변수가 없는 생성자
- 클래스에 생성자가 하나도 없을 경우 -> 자바 컴파일러가 매개변수가 없고, 작동하는 코드가 없는 기본 생성자를 자동으로 생성
- 생성자가 하나라도 있을 경우 -> 자바는 기본 생성자를 만들지 않음

### 5. 생성자 - 오버로딩과 this()
- 생성자 : 메서드 오버로딩처럼 매개변수만 다르게 해서 여러 생성자 제공 가능
- `this()` : 생성자 내부에서 자신의 생성자 호출 가능
  - 생성자 코드의 첫줄에만 작성 가능
- `this` : 인스턴스 자신의 참조값
